# mcp-snippetslab

**MCP server for [SnippetsLab](https://snippetslab.app)** вҖ” connects AI assistants to your personal code snippet library via the [Model Context Protocol](https://modelcontextprotocol.io/). Search across hundreds of snippets, browse by folder or tag, read full content with syntax highlighting info, and create new snippets вҖ” all from your AI assistant.

[**SnippetsLab**](https://snippetslab.app) (by Renfei Song) is a full-featured snippet manager and noteвҖ‘taking app for developers on macOS and iOS. It supports syntax highlighting for 600+ languages, Markdown with Mermaid diagrams and LaTeX, iCloud sync, and automatic backups.

> **Does SnippetsLab have built-in MCP support?** No. SnippetsLab does not expose a Model Context Protocol interface as of version 2.6.4 (February 2026). This server bridges that gap by reading from SnippetsLab's automatic backup files and writing directly to its iCloud library.

## Key Features

- **Read from backups** вҖ” parses SnippetsLab's automatically created backup files (`library.json`), always available with ~daily freshness
- **Write to the live library** вҖ” creates new snippets as `NSKeyedArchiver` binary plists directly in the SnippetsLab iCloud library bundle; SnippetsLab picks them up automatically
- **6 MCP tools** вҖ” list, search, get, and create snippets; list folders and tags
- **4 MCP resources** вҖ” access snippets, folders, and tags via URI (`snippetslab://`)
- **macOS only** вҖ” SnippetsLab runs on macOS (and iOS), and the iCloud library path is macOS-specific

## How It Reads & Writes SnippetsLab Data

### Reading (backup files)

SnippetsLab automatically creates daily backups at:

```
~/Library/Containers/com.renfei.SnippetsLab/Data/Library/Application Support/Backups/
  <date>.snippetslab-backup/library.json
```

The backup is pure JSON containing all snippets, folders, and tags inline. The server finds the latest backup directory (sorted by name) and parses `library.json`. Results are cached with a 60-second TTL.

### Writing (live iCloud library)

SnippetsLab stores its live data in an iCloud-synced bundle at:

```
~/Library/Mobile Documents/iCloud~com~renfei~SnippetsLab/
  main.snippetslablibrary/Database/Snippets/<UUID>.data
```

Each `.data` file is an `NSKeyedArchiver` binary plist containing a dictionary with SnippetsLab's custom keys. The server constructs the same dictionary structure and writes it as a binary plist. SnippetsLab monitors this directory via `FSEvents` (macOS file system events) and picks up new snippets automatically.

> **Note:** SnippetsLab uses custom Objective-C classes (`SLSnippet`) for its archived objects. These cannot be decoded by `NSKeyedUnarchiver` without the SnippetsLab app running. The server sets `requiresSecureCoding = false` to work around this вҖ” for both reading the backup (fails gracefully) and writing new snippets.

- **Application** вҖ” MCP handler registration (`MCPServerConfiguration`)

- **Composition Root** вҖ” `main.swift` wires everything together

## Quick Start

```bash
# Clone and build
git clone https://github.com/teran/mcp-snippetslab.git
cd mcp-snippetslab
swift build -c release
```

Add to your MCP client configuration:

```json
{
  "mcpServers": {
    "mcp-snippetslab": {
      "type": "local",
      "enabled": true,
      "command": ["/path/to/mcp-snippetslab/.build/release/mcp-snippetslab"]
    }
  }
}
```
